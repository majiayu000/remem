- `https://majiayu000.github.io/remem/codex-memory/`

The live sitemap lists all four URLs, and each URL returned HTTP 200 on 2026-06-19.

**Closed issue:** #538 - https://github.com/majiayu000/remem/issues/538.

**Merged PR:** #539 - https://github.com/majiayu000/remem/pull/539.

**Remaining work:** Search Console submission/request-indexing is tracked in #554.

### S2 - Broad brand phrase collides with an unrelated "Remem AI" product

**Fact:** Search results for broad `Remem AI` terms include an unrelated iOS/Product Hunt product. This repo already uses stronger phrases like "Persistent memory for Claude Code and OpenAI Codex", but the site and registries should keep reinforcing the developer-tool identity.

**Impact:** Users searching from memory may land on the wrong "Remem AI" unless snippets include `Claude Code`, `Codex`, `MCP`, `Rust CLI`, and `coding agent memory`.

**Done when:** Homepage/schema/package copy consistently uses disambiguating phrases; JSON-LD `sameAs` includes GitHub, npm, crates.io, docs.rs/lib.rs; README first screen and package descriptions avoid generic "AI memory" alone.

**Owner issue:** #552 - https://github.com/majiayu000/remem/issues/552.

### S3 - Cargo homepage differs from npm and site canonical

**Fact:** npm homepage points to `https://majiayu000.github.io/remem/`, while `Cargo.toml` homepage points to the GitHub repository.

**Impact:** crates.io/lib.rs users and search crawlers get a weaker product-site signal than npm users.

**Done when:** Cargo homepage points to the product site, repository remains the GitHub URL, and `cargo publish --dry-run --locked` is verified before release.

**Owner issue:** #553 - https://github.com/majiayu000/remem/issues/553.

### S4 - Search Console follow-through is now the main indexing task

**Fact:** The site has a Google verification file and crawlable sitemap. The long-tail pages are live, but Google indexing still depends on Search Console submission/request-indexing and crawler timing.

**Impact:** The repository can ship SEO pages while search indexing lags or is never manually requested.

**Done when:** Submit the updated sitemap in Search Console, request indexing for the homepage and four long-tail URLs, and record the submission date/status.

**Owner issue:** #554 - https://github.com/majiayu000/remem/issues/554.

### S5 - Registry/package smoke should be part of SEO release proof

**Fact:** npm and crates surfaces are discoverability surfaces. The package is currently live, but future metadata changes should prove the install and version path still works after publishing.

**Impact:** SEO metadata can look correct while the package install path is broken, hurting conversion from search traffic.

**Related engineering issue:** #550 - https://github.com/majiayu000/remem/issues/550.

**Done when:** Any package metadata release verifies `npm pack`/install/`remem --version`; crates changes verify `cargo publish --dry-run --locked`.
