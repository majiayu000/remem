## Design Rules

- The new field must be optional so historical golden fixtures remain valid.
- Associative validation must be enforced by the loader, not by comments.
- Query-target overlap uses deterministic normalized ASCII alphanumeric tokens
  with tokens of length three or more; acceptance for this slice is zero shared
  tokens.
- The report must not imply that literal traversal, per-channel attribution,
  or ADR re-recording has happened.
- No runtime schema, hook, capture, API, or production retrieval behavior
  changes are part of this slice.

## Rollback

Remove the additive fixtures, `hop_path` field, associative validation module,
baseline report command, and report artifact. No persisted user data or runtime
behavior is migrated by this slice.
